üîÆ The Prediction Game

Look at this code. If the user inputs 5, what happens?

quantity = int(input("How many numbers? "))

for counter in range(quantity):
    print(counter + 1)

Output:

1, 2, 3, 4, 5

Ask: "Why did we need `counter + 1`? What does `range()` start at by default?"

0Ô∏è‚É£ The Power of Zero

Python starts counting at 0, not 1!

range(5)

Starts at 0

Ends BEFORE 5

0, 1, 2, 3, 4

Result

It runs 5 times.

But the numbers are 0-4.

üïµÔ∏è Task 1.21: The Range Mystery

We want to count numbers between a start and end value.

start = int(input("Start: ")) # e.g., 1
end = int(input("End: ")) # e.g., 10
quantity = 0

for counter in range(start, end + 1):
    quantity = quantity + 1

print("There were", quantity, "numbers")

‚ùì Why `end + 1`?

In the previous code: range(start, end + 1)

Why did we add +1 to the end variable?

üí° The Answer

Because `range()` STOPS before the second number.

If we want to include the number 10, we must tell `range` to stop at 11.

üõ†Ô∏è Modify Challenge

Goal: Change the algorithm so it does NOT include the start or end numbers.

Example: If I enter 1 and 5, I only want to count 2, 3, 4.

# Original: range(start, end + 1)

# Modified:
for counter in range(start + 1, end):
    quantity = quantity + 1

üìò Syntax: Two Types of Range

Type 1: Simple

range(stop)

Starts at 0. Stops before 'stop'. Step is 1.

range(5)

Type 2: Custom

range(start, stop, step)

Control everything!

range(1, 10, 2)

üèÜ Task 1.22: Highest Score

Find the highest score in the class.

highestScore = 0
highestLearner = 0

for learner in range(0, numberOfLearners):
    score = int(input("Enter score: "))
    if score > highestScore:
        highestScore = score
        highestLearner = learner + 1 # Fix index!

Question: Why learner + 1?

Answer: To convert computer counting (0, 1, 2) to human counting (Student 1, Student 2, Student 3).
